These are examples of using Data URI Scheme. Meaning assets can be part of Server-Side-Rendering, which the page could be cached and the asset doesn't have to be requested over the network. We increase the intial payload (HTML source) size a bit, but remove the network call to go get the asset as well as any content shift / delay rendering that happens because of waiting.

We must make sure these are actually used on the initial page load. Like naviagtion logos, or page specific backgrounds.

A CSS class example with the file contents dumped into the attribute, background: url('data:image/svg+xml;utf8, < svg > ... < / svg > ')
